# User Flows

> Define your key product flows here. For each flow, describe every step and
> what "correct" looks like at that step. The agent uses this to evaluate
> whether a screen matches expected behavior and flag deviations.

---

## How to add a flow

```markdown
## [Flow name]

**Entry point:** [where the user starts]
**Exit point:** [what success looks like]

| Step | Screen / State | What correct looks like |
|---|---|---|
| 1 | [screen name] | [what should be visible and functional] |
| 2 | [screen name] | [what should be visible and functional] |
```
