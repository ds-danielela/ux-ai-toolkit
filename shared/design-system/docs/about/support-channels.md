## Design review

![](https://zeroheight-user-uploads.s3.eu-west-1.amazonaws.com/images/TcKhN2pICuKa4NsPtnJApw.png)

### Description

A design review is a milestone within a product development process where you can gather feedback and evaluation against existing design system guidelines and the need requirements. 

You could send a design review request for two purposes:

**Explorative design review**: an early stage review of the basic concept to brainstorm ideas, identify the compliance issues and risks, and clarify the objectives.

**Design system review**: a later stage review focused on final details to ensure the design is ready for production, and consistent with design system standards.

Design reviews operate with different urgency levels, each tied to an expected response time.

### Priority Model

You can use the design review priority model to communicate the expected response time and level of attention with design system team, based on urgency and impact.

**Urgent**: used for the most time‑sensitive requests. It signals that a response is needed as soon as possible. The urgent review request will be prioritized in current or the upcoming sprint.

> When submitting an urgent request, please include the reason and the expected completion date to help the team plan and prioritize effectively.

**Important**: used for time-sensitive requests. It indicates a high-priority request. These reviews are prioritized in the design system's planning but do not require immediate interruption.

**Medium**: used for default design system review requests. Requests are handled on a first‑come, first‑served basis through the standard review planning process. Capacity is limited, with typically 4 Medium‑priority reviews planned per sprint, and additional requests scheduled for future sprints.

**Low**: used for default explorative design review requests. These requests are handled when capacity allows and can be rescheduled further without risk.

To learn more about how the design review process works, please take a [design review process diagram](https://wiki.dentsplysirona.com/display/LIGUI/Designing#:~:text=passed-,Design%20review%20process,-PRODUCT).

## Icon request

![](https://zeroheight-user-uploads.s3.eu-west-1.amazonaws.com/images/Ta6ZwkU0afvpEx3mq1UWWQ.png)

An icon request is a way to propose new or updated icons needed for your product. 

> Before requesting a new icon, please check the [existing icon sets](https://zeroheight.com/0b922d40b/v/0/p/096e14-icons) to see if a suitable option is already available.

When submitting an icon request, please clearly describe:

* Icon usage context
* Why existing icons are not sufficient
* Any research or exploration that has already been done

Icon requests are collected and processed **in batches** rather than individually. 

Each batch is carefully reviewed to assess actual need, avoid duplication, and ensure consistency with the design system. Once approved, the icons are coordinated with our **external design agency**, who produces them to a high visual and technical standard.

Because this workflow involves batching, review cycles, and external collaboration, there is an inherent waiting period before icons are delivered. Based on past experience, this can include up to one month working with the agency and to iterate the design, followed by an additional month to implement the icons in code.

To learn more about how the new icon design process works, please take a look at [icon design process diagram](https://wiki.dentsplysirona.com/display/LIGUI/Designing#:~:text=advice%20on%20alternatives-,Icon%20design%20process,-PRODUCT%20DESIGNER).

## Illustration request

![](https://zeroheight-user-uploads.s3.eu-west-1.amazonaws.com/images/jBQueWcxiiFgf6p4TraJxg.png)

An illustration request is a way to propose new or updated illustrations needed to support your product's communication or storytelling needs.

When submitting an illustration request, please clearly describe:

* Illustration usage context
* Any reference pictures or proposed ideas
* Any research or exploration that has already been done

Illustration requests are reviewed to ensure they are relevant, necessary, and consistent with the overall product and visual language. Approved requests are then coordinated with our **external design agency**, who creates high‑quality illustrations aligned with product needs and design system standards.

Because this process involves review, prioritization, and external collaboration, illustration requests are handled with an inherent waiting period. The exact timeline depends on the illustration's complexity and priority.

To learn more about how the new illustration design process works, please take a look at [illustration design process diagram](https://wiki.dentsplysirona.com/display/LIGUI/Designing#:~:text=Illustration%20design%20process,-PRODUCT).
