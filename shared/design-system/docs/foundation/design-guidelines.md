# Design guidelines

> Source: https://zeroheight.com/0b922d40b/v/latest/p/59d6d0-design-guidelines

Use this checklist when reviewing your designs before handoff or design review.

| | Guideline |
| --- | --- |
| [ ] | **Did you consider Accessibility?** Comply with a minimum of level 2 of the Web Content Accessibility Guideline (WCAG 2.2). |
| [ ] | **Are you asking for only essential information?** Request only essential information from users, clearly distinguishing required fields from optional ones. This approach helps users easily identify what information is necessary, streamlining the process and reducing frustration. |
| [ ] | **Did you consider edge cases in the designs?** When designing, include both standard and alternative paths, considering potential errors. This ensures that users can navigate smoothly even when things go wrong. |
| [ ] | **Did you consider active vs inactive actions?** Show active actions; avoid inactive ones. Display active buttons initially and clearly reveal any additional actions required post-selection. Avoid using disabled buttons without a clear explanation of why it is disabled. |
| [ ] | **Are your designs responsive?** Design with every form factor in mind, ensuring accessibility and functionality across all platforms. By starting with responsive XL layouts and refining down to S, we guarantee that all key screens are optimized for various device sizes. |
| [ ] | **Is communication and tone of voice clear?** Write with the DS Core voice and tone in mind, ensuring clarity and professionalism. By using realistic text and avoiding spelling mistakes, make your messages clear and effective. Double-checking all screens with the UX writer before development guarantees consistency and accuracy. |
| [ ] | **Do you provide a smooth navigation?** Users should always know where they are and how to navigate back. We maintain consistent main menu items across all pages. This navigation follows DS Core information architecture, maintained by the UX&D team. UX approval is required for new navigation menu items. |
| [ ] | **Is the response time optimised?** Design with user response times in mind, ensuring that interactions feel instantaneous and seamless. Recommended response times: <0.1 seconds for immediate feedback / <1.0 second for maintaining flow / <10 seconds for longer tasks (provide ability to continue working while the task in progress). |
| [ ] | **Are the empty states useful to the user?** Ensure that empty states are useful with helpful suggestions and engaging visuals. |
