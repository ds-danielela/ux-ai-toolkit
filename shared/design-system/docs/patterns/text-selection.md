**Text selection: Preview**

---

To allow the user to copy specific data, text should be selectable in general. When the user moves the mouse over text, that is not part of an interactive component (such as a card), the operating system specific text selection cursor occurs. Now the user can select different characters or words by dragging the cursor. The selection is highlighted in a system-specific color. Right-clicking on the selection displays a browser/system specific context menu which allows the user i.e. to copy the selected text.

![](https://zeroheight-user-uploads.s3.eu-west-1.amazonaws.com/images/ePqulHXfF1IbpyQSChn84Q.gif?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=AKIA3AVNYHQK6SJI2PWU%2F20260508%2Feu-west-1%2Fs3%2Faws4_request&X-Amz-Date=20260508T133040Z&X-Amz-Expires=604800&X-Amz-SignedHeaders=host&X-Amz-Signature=a41932361ecfe7d28b5418e1616b1993d88963aebed64bd84a10229cdc841e23)

The selection can be cancelled by clicking again on the selection area.

**Please note:** Due to technical restrictions, it is currently not possible to select a word by double-clicking on it.

**Prevent text selection**

It is also possible to prevent text selection in certain areas of the user interface. However, this should only be done in exceptional cases.

### Do's and Don'ts

Support text selection so that the user is able to copy specific information. This can be crucial especially in tables or page headers.
