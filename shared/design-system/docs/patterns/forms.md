## Usage

Use a form to allow users to enter data, submit information, or configure settings.

Typical use cases are:

* Sign up/login to an account
* Taking a survey
* Placing an order or purchasing a product
* Registering for a service
* Configuring settings (e.g. enabling notifications)
* Providing feedback

### Placement

Forms can appear in multiple areas within the experience. Below are a few examples of forms use cases. Please note that the button position also depends on the usage (also see [common actions](https://zeroheight.com/0b922d40b/p/33bf1c)).

### Form components

Text and control inputs are used to provide users the ability to input or select data into a form. A range of inputs are provided, and each component serves a specific use case.

| **Component** | **Usage** | **Situation** |
| --- | --- | --- |
| [Text field](https://zeroheight.com/0b922d40b/p/523b03) | Enter single-line text | Names, phone numbers, addresses, etc |
| [Text area](https://zeroheight.com/0b922d40b/p/87c9c9) | Enter multiple lines of information or text | Feedback, support requests or detail information |
| [Password field](https://zeroheight.com/0b922d40b/p/15fc33) | Enter private data by initially showing hidden characters | Passwords, PINs, credit card info, Social Security Numbers (SSN) |
| [Dropdown](https://zeroheight.com/0b922d40b/p/509c11) | Select a choice from a pre-determined list | Selecting state, country, or language |
| [Date input](https://zeroheight.com/0b922d40b/p/1214b6) | Enter or select a date | Schedules, meetings, time-based events |
| [Checkbox](https://zeroheight.com/0b922d40b/p/546c9e) | Select or deselect one or multiple items | Agreements and confirmation selections, select all or add options |
| [Radio button](https://zeroheight.com/0b922d40b/p/973639) | Select one option from two or more choices | Selecting a specific type, detail, service, etc. |
| [Switch](https://zeroheight.com/0b922d40b/p/7699c7) | Activate or deactivate an option | On or off, show or hide, changing settings |
| [Slider](https://zeroheight.com/0b922d40b/p/2508f3) | Adjust numerical value on a scale of given values | Selecting percent values, volume, and data visualization |
| [Selectable card](https://zeroheight.com/0b922d40b/p/172ef8) | Select one or more options represented in a detailed way | Selecting a product or service |
| [File upload](https://zeroheight.com/0b922d40b/p/8611c4) | Upload file(s) to a form for submission | Include attachments, files, or tickets |

### Required fields

Only ask the user for information that is really needed. Therefore, there is no need to mark required fields. If in an exceptional case a field is optional, this should be indicated by the addition "(optional)" to the label.

### Validation

A form is validated after the user has pressed the button to submit the form. Error messaging should appear as validation states on the form field, and with an inline notification at the top.

Common errors include:

* Leaving a required input field blank
* Leaving a required input field incomplete
* Incorrectly formatting data

### Content guidelines

* Group similar, related input fields.
* Make it easy for the user to complete the information.
* Use task-specific language for actions ("Create account" instead of "Submit").
* Use a logical order and flow of information (first name first, last name, etc.).
* Only ask the user required information. If the field is optional, this should be indicated by the addition "(optional)" to the label.

### Do's and Don'ts

For clarity on required information, only ask for the required information. Add "optional" to the label when information isn't required.
