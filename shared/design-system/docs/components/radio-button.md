## Usage

Use radio buttons to offer users a choice of at least two options. Only one option can be active at the same time. All options of a radio button group are visible to the user at all times. The state of the radio button group should toggle when clicking on one option or the corresponding label.

As a general rule a radio button group should consist of a maximum of ten options. Also, a radio button group should only be used if the options need to be visible all the time. For more than ten options or if the options can be collapsed, a dropdown should be used.

If the user is also able to select none of the options, add another option at the bottom of the list to reflect this choice, such as "None" or "Does not apply".

### Default state

Only provide a default selection if the user's choice is not critical. Otherwise, do not preselect an option. Once one radio button has been selected, the user cannot return to having no radio button selected.

### Sorting

In a radio button group, list most common options first or follow some other logical order. As it is language depended, alphabetical sorting is not recommended.

### Content guidelines

* Always use clear and concise labels for radio buttons.
* If you are tight on space, consider rewording the label. Long labels may wrap to a second line, and this is preferable to truncation.

### Do's and don'ts

Do not use commas or semicolons at the end of each line.

Use radio buttons for exclusive choices. If the user can select multiple options from a list, use checkboxes instead.

Write in sentence case.
